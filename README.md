# Perfil de Jogos — Gui

> Documento de referência sobre meus gostos em jogos. Estruturado como documentação técnica: separa conclusões (princípios) das evidências que as sustentam, com nível de confiança explícito. Objetivo: recomendações auditáveis, não uma lista de "gostos" soltos.

## Resumo rápido

Atualmente, jogos tendem a funcionar melhor para mim quando combinam:

- mundo interessante e coerente;
- gameplay mecanicamente divertida;
- sensação de aventura completa;
- exploração recompensadora;
- progressão constante.

Tendem a funcionar pior quando dependem principalmente de:

- grind repetitivo;
- endgame infinito;
- repetir o mesmo conteúdo muitas vezes;
- gameplay fraca sustentada apenas pela história.

## Metodologia

Este repositório descreve tendências observadas ao longo do tempo, não regras absolutas.

Alguns princípios:

- Um único jogo raramente é suficiente para confirmar ou invalidar um princípio.
- As conclusões devem ser sustentadas por múltiplas evidências.
- Sempre que possível, registrar o motivo pelo qual gostei ou não de um jogo.
- Jogos recentes têm prioridade sobre lembranças antigas caso haja conflito.
- O perfil deve evoluir conforme novos jogos são concluídos.

## Como interpretar este perfil

Este repositório não busca classificar jogos por gênero.

Em vez disso, procura identificar quais características tornam uma experiência divertida para mim.

Por isso, recomendações devem ser baseadas principalmente nos princípios documentados, e não apenas em jogos semelhantes ou no mesmo gênero.

## Como este repositório está organizado

- `README.md` (este arquivo) — visão geral e resumo do perfil
- `prompt-inicial.md` — protocolo de conversa; colar no início de sessões sobre jogos
- `principios.md` — o que define meu gosto, com relevância e confiança, mais os antipadrões
- `jogos/jogados.md` — jogos que joguei ou abandonei, com nota, impressão e (nos abandonados) o motivo
- `jogos/wishlist.md` — jogos que pretendo jogar, com prioridade e hipótese
- `nao-recomendar.md` — jogos já descartados, com o motivo
- `historico.md` — changelog de mudanças em princípios (uso pessoal, para auditoria)
